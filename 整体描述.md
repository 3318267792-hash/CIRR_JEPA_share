# CIRR-JEPA 项目整体描述

## 1. V0：Baseline 闭环搭建

### 核心思路

V0 的目标是先搭建一个最小可运行闭环，完成从数据读取、target 编码、query 编码，到训练和评测的完整流程。

整体结构为：

- target image -> frozen CLIP-L image encoder -> target_emb
- reference image + modification text -> frozen CLIP -> 简单融合 / MLP -> pred_emb
- 用 pred_emb 去对齐 target_emb

### 主要实现

这一阶段主要完成了三部分工作：

**（1）数据层搭建**

完成了 CIRR 数据集解析、样本读取、图片路径建立以及 DataLoader 基础链路。

核心文件：
- cirr_dataset.py
- scripts/debug_cirr_json.py
- scripts/debug_dataset.py
- scripts/debug_dataloader.py

**（2）Target encoder 搭建**

使用 CLIP-L image encoder 作为 target side，输出固定维度的 target embedding。

核心文件：
- models/clip_target_encoder.py
- scripts/debug_target_encoder.py

**（3）Baseline query encoder 与 train / eval 闭环**

实现 baseline query side，并完成训练与评测脚本。

核心文件：
- models/baseline_query_encoder.py
- scripts/debug_query_encoder.py
- scripts/debug_query_target_pair.py
- utils.py（cirr_collate_fn）
- train_one_step.py
- train_cirr_baseline.py
- eval_cirr_baseline.py

### 阶段结果

V0 阶段已经完成了 baseline 的完整闭环，能够：

- 正常训练
- 保存 checkpoint
- resume 继续训练
- 进行 CIRR eval

### 阶段意义

这一版最重要的意义，不是做性能提升，而是：

- 把整个任务链路先跑通
- 建立后续改进所依赖的 baseline
- 明确数据、训练和评测流程的基本结构

## 2. V1：在 Baseline 上加入 temperature / logit scale

### 核心思路

V0 中使用的是归一化后的 embedding 相似度，直接做：

`pred_emb @ target_emb.T`

由于 cosine similarity 的值域较窄，softmax 后分布不够尖锐，训练信号偏弱，因此在 V1 中主要加入了 可学习的 temperature / logit scale，以增强训练过程中的对比信号。

主要目的包括：

- 放大相似度 logits
- 增强交叉熵训练信号
- 不改变整体 retrieval 的排序逻辑

### 主要实现

这一阶段没有改动整体框架，而是在 baseline 上做小增强。

核心改动包括：

- 在 query encoder 中加入 logit_scale
- 训练时改为 `scale * (pred_emb @ target_emb.T)`
- 保留原有 train / eval 主体逻辑

涉及文件主要是：
- models/baseline_query_encoder.py
- train_cirr_baseline.py

### 阶段意义

V1 的重点不是改架构，而是在原 baseline 上做一个更规范的对比学习训练增强。

可以把这一版理解为一个：

低风险、小改动的性能强化版本。

## 3. V2：Attention 融合版本

### 核心思路

在 V0 / V1 中，query side 的融合方式本质上仍然比较简单：

- 用 reference image embedding 作为主干
- 用 text embedding 作为一个整体修正量
- 最后通过简单加法 / MLP 得到预测 embedding

这种方式的局限在于，它默认文本只是在做一个"全局偏移"，并没有真正建模图像和文本之间的交互关系。

因此在 V2 中，主要升级为：

- 不再只做简单加法
- 把 ref_emb 和 text_emb 当作两个 token
- 用一个小的 attention / transformer block 做融合
- 让模型自己学习图像信息保留多少、文本信息注入多少，以及二者如何交互

### 主要实现

这一版的主要改动集中在 query side。

核心文件：
- models/attention_query_encoder.py（或对应 attention 版 query encoder 文件）
- train_cirr_baseline.py
- eval_cirr_baseline.py

训练和评测流程整体仍然沿用 baseline 框架，重点是把 query encoder 从简单 MLP 融合升级为 attention 融合。

### 阶段结果

这一版相比 baseline 和后面的 qwen 初版，效果更好。

### 阶段结论

这一版带来的主要结论是：

- query side 的融合能力确实是当前主要瓶颈之一
- 保持 target side 稳定、优先增强 query side 这条主线是合理的

## 4. V3：Qwen3-VL 作为更强 query encoder

### 核心思路

在 V2 中，attention 融合虽然增强了 query side，但输入仍然只是 CLIP 提供的 pooled image / text embedding，表达能力仍然有限。

因此 V3 的目标是进一步增强 query side，让 query side 的主体直接由 Qwen3-VL 承担。

主要思路是：

- 输入 reference image + modification text
- 先组织成 Qwen 可接收的多模态输入
- 图文融合主要在 Qwen 主干内部完成
- 从 Qwen hidden states 中取出 query 表示
- 再通过 projection head 映射到 CLIP target space

整体结构为：

- target image -> frozen CLIP-L -> target_emb
- (ref_img, text) -> Qwen3-VL -> hidden states -> pooling -> projection -> pred_emb
- `loss = CE(scale * pred_emb @ target_emb.T)`

### 主要实现

这一版的主要改动是重写 query side。

核心文件：
- models/qwen_query_encoder.py
- train_cirr_baseline.py（或对应训练脚本中的 Qwen 接入部分）
- eval_cirr_baseline.py

此外，在服务器运行和训练效率方面，还补充了：

- 本地下载 HF 模型
- 修改本地 / 服务器模型路径
- 调整 DataLoader 配置（如 num_workers、persistent_workers、prefetch_factor）

### 阶段结果与问题

这一版已经完成了初步训练和评测尝试，但当前结果明显不如 attention 版本。

因此目前的判断是：

- 仅仅把 query side 换成 Qwen，并不会自然得到更好的效果
- 当前主要瓶颈很可能在于 Qwen query representation 到 CLIP target space 的映射过难
- batch size、负样本规模、DataLoader 配置会带来一定影响，但不是根本原因
- 当前更像是 结构问题 / 空间对齐问题，而不只是训练时间不够的问题

## 5. 下一步拟推进方向（当前仍处于设计阶段）

在当前 Qwen 版本效果不理想之后，下一步的主线设计已经初步明确，但还没有正式开始完整实现。

### 核心思路

目前认为问题不只是 Qwen 本身，而更可能在于：

- 现有 readout 方式过于粗糙
- query side 输出到 target side 的空间映射不够稳定
- 只有 batch 内 CE，不足以约束空间几何结构

因此下一步准备从以下几个方面继续推进。

**Query side**
- Qwen3-VL
- output_hidden_states=True
- 取最后 K 层 hidden states
- learned weighted sum
- attention pooling / caption-span pooling
- query projector
- 输出 z_q

**Target side**
- frozen CLIP-L
- pooled target_emb
- target projector
- 输出 z_t

**Loss**
- L_ce(z_q, z_t)
- 再加一个 pair alignment loss：L_align(z_q, z_t)
- 最终为：`L = L_ce + λ * L_align`

### 当前状态

这一版目前还处于设计和主线确认阶段，尚未正式开始完整代码实现。

它代表的是下一阶段准备推进的方向，而不是已经完成的结果。
